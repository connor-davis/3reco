# Setup & Installation

This guide walks you through getting 3rEco running locally.

## Prerequisites

| Requirement | Version | Notes |
|---|---|---|
| [Bun](https://bun.sh) | ≥ 1.0 | Primary package manager & runtime |
| [Convex account](https://convex.dev) | — | Free tier available |
| Node.js | ≥ 18 | Needed for some tooling |
| Git | any | For cloning |

## Clone the Repository

```bash
git clone https://github.com/yourusername/3reco.git
cd 3reco/3reco
```

## Install Dependencies

```bash
bun install
```

## Configure Environment Variables

Create a `.env.local` file in the `3reco/` directory:

```bash
# 3reco/.env.local
VITE_CONVEX_URL=https://<your-deployment>.convex.cloud
```

You obtain the `VITE_CONVEX_URL` from your Convex project dashboard (see below).

<Callout type="warning" title="Never commit .env.local">
  The `.env.local` file contains your Convex deployment URL. It is already in `.gitignore` — keep it that way.
</Callout>

## Set Up Convex

### 1. Create a Convex project

1. Sign in at [dashboard.convex.dev](https://dashboard.convex.dev)
2. Click **New project** and give it a name (e.g. `3reco-dev`)
3. Copy the **Deployment URL** — it looks like `https://abcd1234.convex.cloud`

### 2. Deploy schema & functions

```bash
bunx convex dev
```

This command:
- Authenticates with your Convex account
- Pushes all functions from `convex/` to your deployment
- Watches for changes and hot-reloads

### 3. Configure authentication

3rEco uses `@convex-dev/auth` for authentication. After running `convex dev`, navigate to your Convex dashboard, then **Settings → Authentication** and configure your sign-in methods (e.g. email/password, Google OAuth).

## Start the Dev Server

Open a **second terminal** alongside `convex dev` and run:

```bash
bun run dev
```

The app will be available at `http://localhost:5173`.

<Callout type="tip" title="Two terminals">
  You need both `bunx convex dev` (backend) and `bun run dev` (frontend) running simultaneously during development.
</Callout>

## Build for Production

```bash
bun run build
```

Output is placed in `dist/`. Deploy the `dist/` folder to any static hosting provider (Vercel, Netlify, Cloudflare Pages, etc.).

## Available Scripts

| Command | Description |
|---|---|
| `bun run dev` | Start Vite development server |
| `bun run build` | Type-check and build for production |
| `bun run lint` | Run ESLint |
| `bun run preview` | Preview production build locally |
| `bunx convex dev` | Start Convex backend in watch mode |
| `bunx convex deploy` | Deploy Convex functions to production |
